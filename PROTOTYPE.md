# 一、全局框架

## 1.1 全局导航栏 (Top Navigation)

始终固定在页面顶部，包含以下元素：

| 元素 | 说明 |
|------|------|
| **Logo** | Racing Arcade 品牌 Logo，点击返回首页 |
| **导航菜单** | 赛事 / 日历 / 排行榜 / 新闻 |
| **区域切换器** | 下拉选择当前区域（CN / AP / AM / EU），切换后页面内容刷新为该区域数据 |
| **语言切换** | 中文 / English 切换按钮 |
| **通知铃铛** | 未读数量气泡，点击展开最近 20 条通知列表；"查看全部"跳转通知中心（仅登录用户可见） |
| **用户头像/登录** | 已登录：显示头像 + 下拉菜单（我的档案 / 我的赛事 / 我的车队 / 账号设置 / 退出登录）；未登录：显示"登录"按钮 |

## 1.2 全局状态与行为

| 全局行为 | 说明 |
|---------|------|
| **未登录用户** | 可浏览赛事列表/详情/排行榜/日历/新闻/公开档案；点击报名等需登录的操作时弹出登录弹窗 |
| **封禁用户** | 登录后全站顶部显示封禁横幅（含封禁原因和截止时间），禁止报名/抗议等操作，浏览不受限 |
| **信息补全拦截** | 注册后未补全必要信息的用户，全页遮罩层强制补全（常驻国家、语言偏好、主要游玩游戏） |
| **Token 过期** | 前端检测 401 响应，自动跳转登录页并提示"登录已过期" |

## 1.3 页脚 (Footer)

- 平台介绍 / 帮助中心链接 / 隐私政策 / 服务条款
- 社交媒体链接（Discord / Twitter / YouTube）
- MOZA Racing 品牌信息

# 二、页面清单与详细设计

## P1. 登录页 (`/login`)

**入口**：点击导航栏"登录"按钮、需登录的操作触发弹窗

### 页面元素

| 元素 | 说明 |
|------|------|
| 品牌Logo | Racing Arcade Logo |
| 标语 | 平台简介文案 |
| **Pit House SSO 登录按钮** | 主按钮，点击跳转 Pit House OAuth 授权页 |
| **Discord 登录按钮** | 辅助按钮，点击跳转 Discord OAuth 页 |
| **Steam 登录按钮** | 辅助按钮，点击跳转 Steam OpenID 页 |
| 区域/语言信息 | 当前区域和语言提示 |

### 交互逻辑

1. 用户选择一种登录方式 → 跳转对应第三方授权页
2. 授权成功回调本站：
   - **已绑定账号** → 直接登录，跳转首页
   - **新账号** → 跳转注册补全页 (P2)
3. 授权被拒绝 → 返回登录页，无错误提示
4. 第三方服务不可用 → 显示错误提示"登录服务暂时不可用，请稍后再试"

## P2. 注册信息补全页 (`/register/complete`)

**入口**：首次第三方登录成功后自动跳转；或未补全信息的用户登录后强制跳转（不可跳过的遮罩层）

### 页面元素

| 元素 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 昵称 | 文本输入 | 是 | 平台显示名 |
| 头像上传 | 图片上传 | 否 | 默认使用第三方头像 |
| **常驻国家/区域** | 下拉选择 | 是 | 国家列表，选择后自动映射到 CN/AP/AM/EU 区域 |
| **语言偏好** | 单选 | 是 | 中文 / English |
| **主要游玩游戏** | 多选标签 | 是 | ACC / AC / AC Evo / iRacing / LMU / rF2 / ETS2 等 |
| 完成注册按钮 | 按钮 | — | 提交后进入首页 |

### 交互逻辑

1. 所有必填字段校验通过后"完成注册"按钮可点击
2. 提交成功 → 跳转首页
3. 未补全信息关闭页面 → 下次登录强制显示遮罩层，不可跳过

## P3. 首页 (`/`)

### 页面元素

| 区域 | 说明 |
|------|------|

| **排行榜速览** | 总积分 TOP 5 车手列表 |
| **合作伙伴/MOZA品牌** | MOZA 品牌信息和设备展示 |

### 赛事卡片元素（独立赛事）

- 比赛时间

- 车型组 + 子赛事数量
- 下一场赛道名称
- 下一场比赛时间
- 下一场报名人数（X / Y）+ 下一场报名状态标签（右对齐）

- 点击独立赛事卡片 → 跳转独立赛事详情页 (P5)

- 点击新闻条目 → 跳转新闻详情页 (P20)
- 点击排行榜车手 → 跳转车手档案页 (P12)
- 点击"查看全部" → 跳转赛事列表页 (P4)

## P5. 独立赛事详情页 (`/events/{id}`)

仅用于**独立赛事**（不属于任何锦标赛的赛事）。锦标赛子赛事没有独立页面，全部在锦标赛详情页 (P6) 中展示。

### 页面结构（自上而下）

| 区域 | 元素 | 说明 |
|------|------|------|
| **取消横幅** | 红色横幅"本场赛事已取消"+取消原因 | 赛事已取消时显示在页面最顶部 |
| **封面区** | 封面大图 + 赛事名称 + 状态标签（报名中/进行中等） | — |
| **基础信息栏** | 游戏平台、赛道/布局、车型组、天气、是否进站、比赛时间（双时区） | 不直接显示所属区域；比赛时间同时显示赛事当地时区和用户本地时区 |
| **赛制信息** | 练习赛时长、排位赛时长、正赛时长/圈数、赛制规则（富文本）、积分规则（自定义文字 + 积分表格） | — |
| **报名操作区** | 报名按钮 / 已报名状态 / 取消报名按钮 / "尚未开放报名" | Upcoming 状态时不显示报名按钮，显示"尚未开放报名" |
| **报名信息** | 当前报名人数 / 总容量、预计Split数、准入条件描述 | — |
| **Split分组区** | Split 1-N：各Split参赛名单、比赛时间、服务器信息 | 分组公布后显示 |
| **资源下载** | 富文本渲染的资源信息（下载链接、安装说明等） | 公开可见，无需登录；仅在有资源内容时显示 |
| **服务器信息** | 根据游戏类型展示不同内容：**A 类（ACC/AC/AC Evo/LMU/rF2/ETS2 等自建服务器）**展示服务器名称+密码+可选直连链接，加入指引："请在游戏内搜索服务器名称加入"；**B 类（iRacing 官方服务器）**展示 Session 名称+密码或 Hosted Session 链接，加入指引："请在 iRacing 客户端中查找对应 Session"；如管理员未填写则显示"比赛进入方式稍后提供" | 仅报名后可见 |
| **直播区** | Twitch/YouTube嵌入式播放器 | 赛事进行中显示 |
| **参赛名单** | 完整已报名车手列表（头像+昵称+区域） | — |
| **成绩区** | 排名、车手、车队、完赛时间、最快圈、与领先差距、积分 | 成绩发布后显示 |
| **直播回放** | "观看直播回放"按钮 | 赛后显示 |
| **赛事公告** | 关联的公告列表 | — |

### 报名按钮交互状态

| 用户状态 | 按钮展示 | 点击行为 |
|---------|---------|---------|
| 未登录 | "登录后报名" | 弹出登录弹窗 |
| 已登录-未报名-名额充足 | "立即报名" | 进入报名流程 |
| 已登录-未报名-名额已满 | "名额已满-加入候补" | 加入候补名单 |
| 已登录-已报名 | "已报名 ✓"+取消报名按钮 | 显示报名详情和取消选项 |
| 已登录-被禁赛 | "当前无法报名"（灰色） | 提示查看账号状态 |
| **赛事尚未开放报名** | "尚未开放报名"（灰色文字） | Upcoming 状态，不显示报名按钮 |
| **Upcoming 状态重要时间** | 仅显示比赛时间，不显示报名截止时间 | — |
| 报名已截止 | "报名已截止"（灰色） | 不可点击 |

### 报名流程

1. 点击"立即报名"
2. 检查游戏ID/绑定状态 → 缺少则弹出补充引导（填写iRacing ID 或 引导绑定Steam）
3. 检查准入条件 → 若需规则确认：弹出规则内容+勾选框
4. 确认报名 → 成功：显示报名成功+添加日历按钮+分组待公布提示
5. 跨区域赛事：额外提示"该赛事位于XX区域，请注意时区差异"

### 取消报名流程

1. 点击"取消报名"
2. 检查是否在取消报名截止时间前 → 已过期：提示联系管理员
3. 弹出确认对话框 → 确认 → 名额释放

### 其他交互

- **添加到日历**：点击按钮 → 下载 .ics 文件（包含赛事名称、时间、赛道、服务器信息）
- **成绩展示**：成绩表格含名次、车手（可点击→档案页）、车队（可点击→车队页）、完赛时间、最快圈、差距、状态（完赛/DNF/DSQ等）、积分
- **多Split赛事**：Tab切换各Split成绩
- **提交抗议**：赛后一定时间内，成绩区显示"提交抗议"按钮 → 跳转抗议提交页 (P15)

## P6. 锦标赛详情页 (`/championships/{id}`)

锦标赛详情页是锦标赛及其**所有子赛事的统一展示页面**。子赛事没有独立页面，全部在此页内展示。

### 页面结构（自上而下）

| 区域 | 说明 |
|------|------|
| **页面布局** | 左侧主内容（含 Tab 切换）+ 右侧固定侧边栏（下一场赛事信息 + 报名操作区） | 侧边栏 sticky 固定 |
| **封面区** | 封面图 + 锦标赛名称 + 状态标签 | — |
| **Tab 切换** | "锦标赛信息" / "赛程" / "成绩" 三个 Tab | 切换后主内容区刷新 |
| **Tab 1: 锦标赛信息** | 描述、游戏平台、车型组、天气/进站、赛制参数、赛制规则（富文本）、积分规则（自定义文字 + 积分表格）、晋级/淘汰规则、准入条件、资源下载、积分榜 | 不直接显示所属区域 |
| **Tab 2: 赛程** | 按时间顺序展示所有子赛事：未来赛事（含状态标签区分"未来"/"报名中"，"未来"状态不显示报名按钮，"报名中"含报名按钮、已报名显示服务器信息）+ 过往赛事（不含状态标签，"查看成绩"跳转至成绩 Tab） | 默认展开未来赛事、折叠过往赛事 |
| **Tab 3: 成绩** | 统一展示各历史赛事的成绩，支持下拉选择具体赛事、按环节（正赛/排位赛）筛选 | 必须选择具体赛事，无"全部赛事"选项；切换至成绩 Tab 时右侧栏隐藏，成绩区域全宽 |
| **右侧栏·下一场赛事** | 下一场可报名赛事信息：赛事名称、赛道、比赛时间、报名人数/容量（数字独立一行凸显）、进度条、状态标签（人数行右对齐，已报名显示"已报名"，未报名显示赛事状态，不共存）、报名按钮/取消报名按钮、添加到日历按钮、重要时间节点 | sticky 固定，始终可见 |
| **右侧栏·服务器信息** | 报名后显示，独立卡片（绿色边框高亮），位于下一场赛事卡片下方，上下布局（标签/值分行）：根据游戏类型展示对应的服务器/Session信息及加入指引（同P5服务器信息逻辑）；如管理员未填写则显示"比赛进入方式稍后提供" | 仅报名后可见 |
| **锦标赛公告** | 关联的公告列表 |

### "下一场可报名"子赛事区域（默认展开，突出显示）

| 元素 | 说明 |
|------|------|
| 子赛事名称 | 如"第 3 站 - 铃鹿" |
| 赛道 / 布局 | 本场独有 |
| 比赛时间（双时区） | 本场独有 |
| 报名截止时间 | — |
| 报名按钮 / 已报名状态 | 每场子赛事有独立的报名按钮，交互逻辑同独立赛事 |
| 取消报名按钮 | 大小与"添加到日历"按钮一致 |
| 报名人数 / 容量 | "报名人数"标签 + 状态标签右对齐（占一行），数字独立下一行 `text-2xl font-bold` 凸显 + 进度条 |
| 服务器信息（报名后可见） | 独立卡片显示在主卡片下方，绿色边框高亮，上下布局；根据游戏类型展示对应内容及加入指引；未填写时显示"比赛进入方式稍后提供" |
| 直播区域（赛事期间展示） | 优先使用本场直播链接，无则使用锦标赛通用直播链接 |
| 参赛名单 | — |

### "未来赛事"区域（默认展开）

每个子赛事卡片展示：
- 子赛事名称（如"第 4 站 - 蒙扎"）
- 赛道 / 布局
- 比赛时间
- 报名人数
- 不显示状态标签、报名截止时间、报名按钮

### "过往赛事"区域（默认折叠）

每个子赛事卡片展示：
- 子赛事名称、赛道、比赛时间
- 成绩摘要（冠军 / 领奖台）
- **"查看成绩"按钮**：点击后自动切换到"成绩" Tab 并筛选到该赛事
- 直播回放链接（如有）
- 提交抗议入口（赛后规定时间内）
- **不显示状态标签**（过往赛事必然已结束）
- **不再内联展开成绩表**

### "成绩" Tab

统一展示锦标赛内各历史赛事的成绩，方便快速对比。

| 元素 | 说明 |
|------|------|
| **赛事筛选** | 自定义下拉选择框（非原生 select），必须选择一场具体赛事，无"全部赛事"选项 |
| **环节切换** | 正赛 / 排位赛 两个按钮切换，排位赛按最快圈排序 |
| **成绩表** | 排名、车手（可点击→档案页）、车队、总用时（正赛）、完成圈数、最快圈、与领先差距（正赛）、状态、积分 |

### 子赛事展示规则

- 子赛事仅展示本场独有字段：名称、赛道、布局、比赛时间、报名人数、服务器信息（已报名可见，绿色边框高亮）、成绩、抗议入口
- 公共信息（游戏、车型、赛制等）由锦标赛页面顶部统一展示，子赛事区域不重复
- 下一场未开赛的子赛事在右侧栏突出显示（含 RegistrationOpen 和 RegistrationClosed 状态，跳过 Upcoming 状态）
- 未来赛事和过往赛事默认折叠，用户可展开查看详情

# 四、核心交互流程

## 4.1 用户登录 → 浏览 → 报名完整流程

```
登录页(P1) → 第三方授权 → 注册补全(P2)[首次] → 首页(P3)
  → 赛事列表(P4)/日历(P7)

  [独立赛事路径]
  → 独立赛事详情(P5) → 报名 → 参赛 → 查看成绩 → [异议?] → 提交抗议(P13)

  [锦标赛子赛事路径]
  → 锦标赛详情(P6) → "下一场可报名"区域 → 报名 → 参赛 → 过往赛事展开 → 查看成绩
```

## 4.2 报名通用流程（独立赛事 & 锦标赛子赛事通用）

```
点击"立即报名"
  → [缺少游戏ID?] → 弹窗补充 → 填写/绑定
  → [有准入条件?] → 弹窗确认规则
  → 确认报名 → 成功：报名成功+添加日历+分组待公布
  → 跨区域赛事：提示时区差异
```

## 4.3 车队创建 → 邀请 → 报名流程

```
我的车队(P16) → 创建车队 → 填写信息 → 创建成功
  → 邀请成员 → 搜索用户 → 发送邀请
  → 成员接受邀请通知 → 加入车队
  → 独立赛事详情(P5)/锦标赛详情(P6) → 队长报名 → 勾选参赛成员 → 报名成功
```

## 4.4 抗议 → 裁决 → 申诉流程

```
独立赛事详情(P5)成绩区 / 锦标赛详情(P6)过往赛事展开区
  → 提交抗议(P13) → 填写抗议信息 → 提交
  → 管理员裁决 → 双方收到通知
    → [被处罚方不服?] → 提出申诉(P15) → 填写申诉理由
    → 管理员复审 → 最终裁决通知
```

## 4.5 候补转正流程

```
赛事详情(P5)/锦标赛详情(P6) → 点击"名额已满-加入候补" → 进入候补名单
  → 等待有人取消报名 → 收到候补转正通知
  → 限定时间内确认参赛 → 报名成功
  → 超时未确认 → 名额顺延下一位
```

## 4.6 账号绑定/解绑流程

```
账号设置(P10) → 第三方账号Tab
  → 绑定Discord/Steam → 跳转授权页 → 回调绑定
  → 解绑 → [仅剩一种?] → 禁止，提示先绑定其他
           → [多种方式?] → 确认解绑 → 成功
```

# 五、弹窗/浮层清单

| 弹窗 | 触发场景 | 内容 |
|------|---------|------|
| **登录弹窗** | 未登录用户点击需登录操作 | 同登录页内容（Pit House/Discord/Steam按钮） |
| **游戏ID补充弹窗** | 报名时缺少游戏ID | iRacing ID填写框 或 Steam绑定引导 |
| **规则确认弹窗** | 报名有准入条件的赛事 | 赛事规则内容 + 勾选框 + 确认按钮 |
| **报名成功弹窗** | 报名成功后 | 成功提示 + 添加日历按钮 + 分组待公布提示 |
| **取消报名确认弹窗** | 点击取消报名 | 确认对话框 |
| **跨区域提示弹窗** | 报名非本区域赛事 | "该赛事位于XX区域，请注意时区差异" |
| **创建车队弹窗** | 点击创建车队 | 车队名称/Logo/简介/区域表单 |
| **邀请成员弹窗** | 队长点击邀请 | 搜索用户输入框 + 发送按钮 |
| **移除成员确认弹窗** | 队长移除成员 | 确认对话框（提示是否影响进行中赛事） |
| **解散车队确认弹窗** | 队长点击解散 | 确认对话框（有进行中赛事时禁止） |
| **申诉提交弹窗** | 被处罚方点击提出申诉 | 申诉理由 + 补充证据 + 提交按钮 |
| **封禁提示横幅** | 被封禁用户登录 | 封禁原因 + 截止时间（全页顶部横幅） |
| **信息补全遮罩** | 未补全信息用户登录 | 全屏不可跳过，强制填写（常驻国家/语言/游戏） |
